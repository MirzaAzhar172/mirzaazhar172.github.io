Vigilance Monitor: AI Vision + MQTT Alert Infrastructure

Date: June 2025

An intelligent vigilance monitoring system that combines computer vision with IoT messaging infrastructure to detect drowsiness and fatigue in real-time through webcam analysis.

The system processes facial landmarks including Eye Aspect Ratio (EAR) and lip distance measurements, publishing alert data via MQTT protocol to Node-RED automation flows. When drowsiness thresholds are exceeded, automated notifications and alarm systems are triggered through the integrated IoT infrastructure.

This solution provides seamless integration between AI-powered computer vision analysis and robust MQTT messaging for scalable safety monitoring applications across various environments.

GTIDS24 x EvoCity ‚Äì Anti-Microsleep App Flow

Date: July 2025

This flowchart summarizes the full process of the mobile app developed under the GTIDS24 project collaboration between UiTM and EvoCity. The app uses camera-based face detection and AI avatars to monitor driver drowsiness, yawns, and microsleep events in real-time.

After launching the app, users can choose from 7 languages and avatar voices. A manual guide is shown for first-time users. Once activated, the avatar-enabled detector screen begins analyzing user expressions via the camera.

Detected signs (yawn, drowsiness, microsleep) are logged into a dashboard. Each trigger displays a popup and plays a TTS warning in the selected language. All data ‚Äî including time, type, and count ‚Äî is stored and later visualized in charts or exported as a PDF driving activity report.

This project was deployed during GTIDS24, focusing on real-time safety systems for drivers. I served as a Research Assistant during this development, and contributed to UI/UX flows, backend data logging, and testing under real driving conditions.

What is OpenCTI?

Date: July 2025

OpenCTI (Open Cyber Threat Intelligence Platform) is an open-source platform designed to manage and share cyber threat intelligence (CTI). It provides structured and contextualized information such as indicators of compromise (IOCs), attack patterns, threat actors, malware, and more.

During my internship, I used OpenCTI to monitor live threat feeds and correlate data from different sources. The platform supports STIX/TAXII, integrates with MITRE ATT&CK, and helps visualize relationships between threats and assets. It is essential for any SOC team looking to automate, analyze, and share threat intelligence effectively.

Maltrail and How It Works with Arkime

Date: July 2025

Maltrail is an open-source malicious traffic detection system that utilizes publicly available blacklists and machine learning to detect threats in real-time. It logs suspicious DNS queries, IPs, and unusual behaviors.

In my setup, I integrated Maltrail with Arkime (formerly Moloch) ‚Äî a large-scale packet capture and indexing tool. Arkime captures and indexes network traffic, while Maltrail flags suspicious indicators. By correlating alerts from Maltrail inside Arkime's search interface, we can trace back full packet logs and better understand the nature of the threat.

This integration enhances SOC visibility and enables deep packet inspection on malicious activities detected by Maltrail, improving response time and threat analysis.

Microsleep Detection System - IoT Prototype

Date: June 2025

üîß Project Highlights:

  • Developed using Python, dlib, OpenCV, Node-RED, Slack API, MQTT, and MongoDB.
  • Real-time facial landmark tracking and detection using x68 landmarks and Haar Cascade.
  • Speech alerts via eSpeak (e.g., "Danger!! WAKE UP!" and "Stop Driving! You need to take some fresh air.")
  • Live data published from Python to Node-RED using MQTT for dashboard visualization.

üéØ Objectives:

  • Identify physiological indicators of driver drowsiness and microsleep.
  • Develop an IoT-based anti-microsleep alarm for drivers.
  • Test the usability and effectiveness of the system.

üí° Problem Statement:

  • Current drowsiness detection systems lack reliability and personal adaptability.
  • There is a need for early warning systems with configurable multi-modal alerts.
